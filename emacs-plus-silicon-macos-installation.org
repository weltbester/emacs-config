#+author: M365 Copilot

* Vorbemerkung

Diese Anleitung beschreibt die Installation einer Emacs-Plus-Umgebung auf einem Apple-Silicon-Mac mit Homebrew unter =/opt/homebrew=.

Sie ist bewusst als Installationsanleitung formuliert und endet mit einem vollständig überprüfbaren Start beider Betriebsarten:

- direkter GUI-Start ohne Daemon
- Start als benannter Daemon =power= plus GUI-/Terminal-Client via =emacsclient=

Die eigentliche Emacs-Konfiguration, also der vollständige Inhalt von =early-init.el= und =init.el=, sollte separat dokumentiert werden. Diese Datei enthält aber alle Installations- und Startvoraussetzungen, damit Emacs bis zum finalen Start lauffähig ist.

* 1. Bestehende Emacs-Konfiguration sichern

** Installationsanweisung

Vorhandene Emacs-Konfigurationen prüfen:

#+begin_src shell
ls -ld ~/.emacs ~/.emacs.d ~/.config/emacs 2>/dev/null
#+end_src

Backup-Verzeichnis anlegen:

#+begin_src shell
mkdir -p ~/emacs-backups
BACKUP_DIR="$HOME/emacs-backups/emacs-backup-$(date +%Y%m%d-%H%M%S)"
mkdir -p "$BACKUP_DIR"
echo "$BACKUP_DIR"
#+end_src

Vorhandene Konfigurationen sichern:

#+begin_src shell
[ -e ~/.emacs ] && cp -a ~/.emacs "$BACKUP_DIR/"
[ -e ~/.emacs.d ] && cp -a ~/.emacs.d "$BACKUP_DIR/"
[ -e ~/.config/emacs ] && cp -a ~/.config/emacs "$BACKUP_DIR/"
#+end_src

Separate Konfigurationsverzeichnisse anlegen:

#+begin_src shell
mkdir -p ~/.config/emacs-minimal
mkdir -p ~/.config/emacs-power
#+end_src

Minimal-Fallback-Konfiguration anlegen:

#+begin_src shell
cat > ~/.config/emacs-minimal/init.el <<'EOF'
;;; init.el --- Minimal fallback Emacs config

(setq inhibit-startup-screen t)

(menu-bar-mode 1)
(tool-bar-mode -1)
(scroll-bar-mode -1)

(setq make-backup-files nil)
(setq auto-save-default nil)

(message "Loaded minimal fallback Emacs config")
EOF
#+end_src

Power-Config-Platzhalter anlegen:

#+begin_src shell
cat > ~/.config/emacs-power/init.el <<'EOF'
;;; init.el --- Power-user Emacs config placeholder

(setq inhibit-startup-screen t)

(message "Loaded power-user Emacs config placeholder")
EOF
#+end_src

** Erläuterungen / Erklärungen

Es werden zwei getrennte Emacs-Konfigurationsverzeichnisse verwendet:

#+begin_example
~/.config/emacs-minimal
~/.config/emacs-power
#+end_example

Damit kann die Power-User-Konfiguration getestet werden, ohne eine eventuell vorhandene Standardkonfiguration zu überschreiben. Im Fehlerfall kann Emacs weiterhin mit der Minimal-Konfiguration gestartet werden.

** Überprüfen

#+begin_src shell
find "$BACKUP_DIR" -maxdepth 2 -print
find ~/.config/emacs-minimal ~/.config/emacs-power -maxdepth 1 -type f -print
#+end_src

** Erwartete Ausgabe

Mindestens:

#+begin_example
/Users/<user>/.config/emacs-minimal/init.el
/Users/<user>/.config/emacs-power/init.el
#+end_example

Falls vorher bereits Emacs-Konfigurationen existierten, sollten diese zusätzlich im Backup-Verzeichnis erscheinen.

* 7. Native-Compilation-Umgebung vorbereiten

* 20. Emacs-Daemon-Grundtest ohne Shell-Funktionen

** Installationsanweisung

#+begin_src shell

  --init-directory "$HOME/.config/emacs-power" \
  --daemon=power \

disown
#+end_src

#+begin_src shell

#+end_src

#+begin_src shell

#+end_src

#+begin_src shell

* 21. Shell-Funktionen für Daemon, GUI, Terminal und Fallback einrichten

** Installationsanweisung

Zsh-Konfigurationsdatei für interaktive Funktionen anlegen:

#+begin_src shell
touch ~/.zshrc
#+end_src

Folgende Funktionen in =~/.zshrc= eintragen:

#+begin_src shell
cat >> ~/.zshrc <<'EOF'

# Emacs Plus power-user setup - daemon/client helpers

emacs-power-socket() {
  printf "%s/emacs%s/power" "${TMPDIR%/}" "$(id -u)"
}

emacs-power-start() {
  local sock
  sock="$(emacs-power-socket)"

  if [ -S "$sock" ] && emacsclient -s "$sock" -e '(emacs-pid)' >/dev/null 2>&1; then
    echo "Emacs power daemon already running: $sock"
    return 0
  fi

  /Applications/Emacs.app/Contents/MacOS/Emacs \
    --init-directory "$HOME/.config/emacs-power" \
    --daemon=power \
    > /tmp/emacs-power-daemon.log 2>&1 &

  disown
}

emacs-power-gui() {
  emacsclient -s "$(emacs-power-socket)" -n -c "$@"
}

emacs-power-term() {
  emacsclient -s "$(emacs-power-socket)" -t "$@"
}

emacs-power-stop() {
  emacsclient -s "$(emacs-power-socket)" -e "(kill-emacs)"
}

emacs-power-restart() {
  emacs-power-stop 2>/dev/null || true
  sleep 1
  emacs-power-start
}

emacs-power-status() {
  pgrep -af Emacs || true
  echo "Socket: $(emacs-power-socket)"
  ls -l "$(emacs-power-socket)" 2>/dev/null || true
}

emacs-power-log() {
  tail -100 /tmp/emacs-power-daemon.log
}

emacs-power-batch-check() {
  /Applications/Emacs.app/Contents/MacOS/Emacs \
    --batch \
    --init-directory "$HOME/.config/emacs-power" \
    --eval '(message "Power config batch check OK")'
}

emacs-power-direct() {
  /Applications/Emacs.app/Contents/MacOS/Emacs \
    --init-directory "$HOME/.config/emacs-power" "$@"
}

emacs-minimal-direct() {
  /Applications/Emacs.app/Contents/MacOS/Emacs \
    --init-directory "$HOME/.config/emacs-minimal" "$@"
}
EOF
#+end_src

Aktuelle Shell neu laden:

#+begin_src shell
source ~/.zshrc
#+end_src

** Erläuterungen / Erklärungen

Diese Funktionen kapseln die relevanten Startarten:

- =emacs-power-direct=: direkter GUI-Start mit Power-Konfiguration, ohne Daemon
- =emacs-minimal-direct=: direkter GUI-Start mit Minimal-Fallback-Konfiguration
- =emacs-power-start=: Start des benannten Daemons =power=
- =emacs-power-gui=: Start eines GUI-Clients gegen den Daemon
- =emacs-power-term=: Start eines Terminal-Clients gegen den Daemon
- =emacs-power-stop=: sauberes Beenden des Daemons

Die Funktionen verwenden explizit den macOS-Socket-Pfad =${TMPDIR}/emacs$(id -u)/power=, weil das stabiler ist als =emacsclient -s power=.

** Überprüfen

#+begin_src shell
type emacs-power-start
type emacs-power-gui
type emacs-power-term
type emacs-power-direct
type emacs-minimal-direct
#+end_src

** Erwartete Ausgabe

Beispiel:

Der empfohlene Normalbetrieb ist Daemon plus Client. Der Daemon hält die Emacs-Sitzung im Hintergrund, während GUI- und Terminal-Clients schnell starten.

Der direkte Start bleibt wichtig für Debugging. Der Minimal-Fallback bleibt wichtig für Recovery.

- =emacs-power-gui= öffnet ein GUI-Fenster.
- Die Shell kehrt sofort zurück.
- Es erscheint keine Meldung wie =can't find socket=.
