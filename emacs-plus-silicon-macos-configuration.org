* Vorbemerkung

Diese Konfigurationsanleitung ergänzt die separate Installationsanleitung:

#+begin_example
emacs-plus-silicon-macos-installation-v2.org
#+end_example

Sie beschreibt die Konfiguration der Power-User-Umgebung unter:

#+begin_example
~/.config/emacs-power/
#+end_example

Die Anleitung ist nach folgendem Schema aufgebaut:

- Installationsanweisung
- Erläuterungen / Erklärungen
- Überprüfen
- Erwartete Ausgabe

Die Anleitung geht davon aus, dass folgende Basis bereits installiert ist:

- Emacs Plus GUI unter =/Applications/Emacs.app=
- Homebrew unter =/opt/homebrew= auf Apple Silicon
- getrennte Config-Verzeichnisse =emacs-minimal= und =emacs-power=
- externe Tools wie =clangd=, =pyright=, =rust-analyzer=, =rust-script=, =libvterm=, =codex= und =claude=

Für Intel macOS müssen Pfade mit =/opt/homebrew= ggf. durch =/usr/local= ersetzt werden.

* 1. Zielstruktur der Emacs-Konfiguration

** Installationsanweisung

Die Power-Konfiguration liegt unter:

#+begin_example
~/.config/emacs-power/
#+end_example

Darin werden mindestens diese Dateien verwendet:

#+begin_example
~/.config/emacs-power/early-init.el
~/.config/emacs-power/init.el
#+end_example

Die Minimal-Fallback-Konfiguration liegt unter:

#+begin_example
~/.config/emacs-minimal/init.el
#+end_example

** Erläuterungen / Erklärungen

=early-init.el= wird sehr früh geladen, noch vor der normalen Package-Initialisierung. Dort gehören frühe Startup-, PATH-, native-comp- und Performance-Einstellungen hinein.

=init.el= ist die Hauptkonfiguration. Dort liegen Package-Setup, Completion, LSP, Org-mode, AI-Workflow, Magit, Theme und Keybindings.

** Überprüfen

#+begin_src shell
ls -l ~/.config/emacs-power/early-init.el ~/.config/emacs-power/init.el
ls -l ~/.config/emacs-minimal/init.el
#+end_src

** Erwartete Ausgabe

#+begin_example
/Users/<user>/.config/emacs-power/early-init.el
/Users/<user>/.config/emacs-power/init.el
/Users/<user>/.config/emacs-minimal/init.el
#+end_example

* 2. Backup vor Konfigurationsänderungen

** Installationsanweisung

Vor jeder größeren Änderung:

#+begin_src shell
cp -a ~/.config/emacs-power ~/.config/emacs-power.before-change.$(date +%Y%m%d-%H%M%S)
#+end_src

** Erläuterungen / Erklärungen

Dadurch kann die komplette Power-Konfiguration schnell zurückgerollt werden. Das ist einfacher und robuster als einzelne Änderungen manuell rückgängig zu machen.

** Überprüfen

#+begin_src shell

#+end_src

** Erwartete Ausgabe

#+begin_example
/Users/<user>/.config/emacs-power.before-change.YYYYMMDD-HHMMSS
#+end_example

* 3. early-init.el erstellen

** Installationsanweisung

Erstelle oder ersetze:

#+begin_example
~/.config/emacs-power/early-init.el
#+end_example

mit folgendem Inhalt:

#+begin_src emacs-lisp
;;; early-init.el --- Early startup tuning

;;; Homebrew / native-comp environment for Apple Silicon macOS

(when (eq system-type 'darwin)
  ;; Prefer Homebrew tools early.
  (let ((brew-bin "/opt/homebrew/bin")
        (brew-sbin "/opt/homebrew/sbin"))
    (setenv "PATH"
            (mapconcat #'identity
                       (delete-dups
                        (append (list brew-bin brew-sbin)
                                (split-string (or (getenv "PATH") "") path-separator t)))
                       path-separator))
    (setq exec-path
          (delete-dups
           (append (list brew-bin brew-sbin) exec-path))))

  ;; Use explicit GNU GCC binaries. Homebrew does not normally provide
  ;; unversioned gcc/g++ because /usr/bin/gcc is Apple Clang.
  (let* ((gcc-candidates (file-expand-wildcards "/opt/homebrew/bin/gcc-[0-9]*"))
         (gxx-candidates (file-expand-wildcards "/opt/homebrew/bin/g++-[0-9]*"))
         (gcc-bin (car (last (sort gcc-candidates #'string<))))
         (gxx-bin (car (last (sort gxx-candidates #'string<)))))
    (when gcc-bin
      (setenv "CC" gcc-bin))
    (when gxx-bin
      (setenv "CXX" gxx-bin)))

  ;; Help libgccjit find the GCC support libraries when native-compiling.
  (let* ((gcc-lib-current "/opt/homebrew/opt/gcc/lib/gcc/current")
         (libgccjit-lib-current "/opt/homebrew/opt/libgccjit/lib/gcc/current")
         (gcc-arch-dirs
          (file-expand-wildcards
           "/opt/homebrew/opt/gcc/lib/gcc/current/gcc/*-apple-darwin*/*"))
         (candidate-paths
          (append
           (list gcc-lib-current
                 libgccjit-lib-current
                 "/Library/Developer/CommandLineTools/SDKs/MacOSX.sdk/usr/lib")
           gcc-arch-dirs))
         (existing-paths
          (split-string (or (getenv "LIBRARY_PATH") "") path-separator t))
         (valid-paths
          (seq-filter #'file-directory-p candidate-paths)))
    (setenv "LIBRARY_PATH"
            (mapconcat #'identity
                       (delete-dups (append valid-paths existing-paths))
                       path-separator))))

;;; Rust / Cargo path

(when (eq system-type 'darwin)
  (let ((cargo-bin (expand-file-name "~/.cargo/bin"))
        (rustup-bin "/opt/homebrew/opt/rustup/bin"))
    (dolist (dir (list cargo-bin rustup-bin))
      (when (file-directory-p dir)
        (setenv "PATH" (concat dir path-separator (getenv "PATH")))
        (add-to-list 'exec-path dir)))))

;;; Package startup behavior

(setq package-enable-at-startup nil)

;;; Native-comp behavior

(setq native-comp-async-report-warnings-errors nil)

;;; Startup performance

(setq gc-cons-threshold most-positive-fixnum
      gc-cons-percentage 0.6)

(setq frame-inhibit-implied-resize t)

(menu-bar-mode 1)
(tool-bar-mode -1)
(scroll-bar-mode -1)

;;; early-init.el ends here
#+end_src

** Erläuterungen / Erklärungen

Diese Datei sorgt dafür, dass Emacs früh genug die korrekten Homebrew-, GCC-, libgccjit-, Rust- und Cargo-Pfade sieht.

Wichtig ist vor allem:

- =/opt/homebrew/bin= steht früh im PATH.
- native-comp verwendet Homebrew-GCC statt Apple Clang.
- =LIBRARY_PATH= enthält die nötigen GCC/libgccjit-Pfade.
- Cargo-Tools wie =rust-script= werden auch in GUI-Emacs gefunden.

Auf Intel macOS müssen die Pfade =/opt/homebrew= entsprechend durch =/usr/local= ersetzt werden.

** Überprüfen

#+begin_src shell
/Applications/Emacs.app/Contents/MacOS/Emacs \
  --batch \
  --init-directory "$HOME/.config/emacs-power" \
  --eval '(message "early-init load test OK")'
#+end_src

** Erwartete Ausgabe

#+begin_example
early-init load test OK
#+end_example

Es sollten keine Lisp-Fehler erscheinen.

* 4. Basis-init.el mit Package-Setup und UI-Grundlagen

** Installationsanweisung

Erstelle oder ersetze:

#+begin_example
~/.config/emacs-power/init.el
#+end_example

mit folgendem Basisinhalt:

#+begin_src emacs-lisp
;;; init.el --- Power-user Emacs base config

;;; Basic startup behavior

(setq inhibit-startup-screen t)
(setq initial-scratch-message nil)

(add-hook 'emacs-startup-hook
          (lambda ()
            (setq gc-cons-threshold (* 64 1024 1024)
                  gc-cons-percentage 0.1)))

(setq custom-file (expand-file-name "custom.el" user-emacs-directory))
(when (file-exists-p custom-file)
  (load custom-file))

;;; macOS behavior

(when (eq system-type 'darwin)
  (setq mac-command-modifier 'meta)
  (setq mac-option-modifier 'none)
  (setq ns-function-modifier 'hyper)
  (setq ns-use-native-fullscreen t))

;;; Backups and autosaves

(setq backup-directory-alist
      `(("." . ,(expand-file-name "backups/" user-emacs-directory))))

(setq auto-save-file-name-transforms
      `((".*" ,(expand-file-name "auto-save/" user-emacs-directory) t)))

(make-directory (expand-file-name "backups/" user-emacs-directory) t)
(make-directory (expand-file-name "auto-save/" user-emacs-directory) t)

;;; Package management

(require 'package)

(setq package-archives
      '(("gnu"    . "https://elpa.gnu.org/packages/")
        ("nongnu" . "https://elpa.nongnu.org/nongnu/")
        ("melpa"  . "https://melpa.org/packages/")))

(package-initialize)

(unless package-archive-contents
  (package-refresh-contents))

(unless (package-installed-p 'use-package)
  (package-install 'use-package))

(require 'use-package)

(setq use-package-always-ensure t)
(setq use-package-always-defer t)
(setq use-package-verbose nil)

;;; macOS shell environment import

(use-package exec-path-from-shell
  :if (memq window-system '(mac ns x))
  :demand t
  :config
  (setq exec-path-from-shell-variables
        '("PATH" "MANPATH" "SHELL" "LANG" "LC_ALL" "LC_CTYPE"))
  (exec-path-from-shell-initialize))

;;; General editing defaults

(setq-default indent-tabs-mode nil)
(setq-default tab-width 4)
(setq require-final-newline t)
(setq ring-bell-function 'ignore)

(global-display-line-numbers-mode 1)
(column-number-mode 1)
(save-place-mode 1)
(savehist-mode 1)
(recentf-mode 1)

(setq recentf-max-saved-items 200)
(setq history-length 200)

(show-paren-mode 1)
(global-auto-revert-mode 1)
(setq global-auto-revert-non-file-buffers t)

;;; Built-in project support

(use-package project
  :ensure nil
  :defer nil)

(message "Loaded power-user Emacs base config")
#+end_src

Dieser Abschnitt richtet die Grundstruktur der Konfiguration ein:

- Package-Archive GNU ELPA, NonGNU ELPA und MELPA
- =use-package= als Konfigurationsmakro
- macOS-PATH-Import über =exec-path-from-shell=
- separate Backup-/Autosave-Verzeichnisse
- Zeilennummern, Verlauf, recentf, save-place

* 5. Completion- und Navigations-Stack konfigurieren

** Installationsanweisung

Folgenden Block an =~/.config/emacs-power/init.el= anhängen:

#+begin_src emacs-lisp
;;; ---------------------------------------------------------------------------
;;; Completion and navigation
;;; ---------------------------------------------------------------------------

(use-package vertico
  :demand t
  :config
  (vertico-mode 1))

(use-package orderless
  :demand t
  :config
  (setq completion-styles '(orderless basic)
        completion-category-defaults nil
        completion-category-overrides '((file (styles basic partial-completion)))))

(use-package marginalia
  :demand t
  :config
  (marginalia-mode 1))

(use-package consult
  :bind
  (("C-s" . consult-line)
   ("C-x b" . consult-buffer)
   ("C-c f" . consult-find)
   ("C-c g" . consult-grep)
   ("C-c r" . consult-ripgrep)
   ("C-c i" . consult-imenu)
   ("C-c p b" . consult-project-buffer)))

(use-package corfu
  :demand t
  :config
  (setq corfu-auto t
        corfu-auto-delay 0.2
        corfu-auto-prefix 2
        corfu-cycle t
        corfu-preview-current nil)
  (global-corfu-mode 1))

(use-package cape
  :after corfu
  :config
  (add-to-list 'completion-at-point-functions #'cape-file)
  (add-to-list 'completion-at-point-functions #'cape-dabbrev))

(use-package which-key
  :demand t
  :config
  (setq which-key-idle-delay 0.5)
  (which-key-mode 1))

(use-package helpful
  :bind
  (("C-h f" . helpful-callable)
   ("C-h v" . helpful-variable)
   ("C-h k" . helpful-key)
   ("C-h x" . helpful-command)))

(use-package vertico-directory
  :ensure nil
  :after vertico
  :bind
  (:map vertico-map
        ("DEL" . vertico-directory-delete-char)
        ("M-DEL" . vertico-directory-delete-word)
        ("C-l" . vertico-directory-up))
  :hook
  (rfn-eshadow-update-overlay . vertico-directory-tidy))

(message "Loaded completion/navigation configuration")
#+end_src

** Erläuterungen / Erklärungen

Dieser Stack ersetzt keine Emacs-Grundfunktionen, sondern verbessert sie:

- =vertico=: vertikale Completion im Minibuffer
- =orderless=: flexible Suchmuster
- =marginalia=: Zusatzinfos im Minibuffer
- =consult=: schnelle Suche und Navigation
- =corfu=: Completion im Buffer
- =cape=: zusätzliche Completion-Quellen
- =which-key=: Keybinding-Hilfe
- =helpful=: bessere Help-Buffer
- =vertico-directory=: ordnerweises Navigieren beim Öffnen von Dateien

Das ordnerweise Navigieren funktioniert im Dateiprompt z. B. mit:

- =M-DEL=: Pfadkomponente löschen
- =C-l=: ein Verzeichnis nach oben

  "Install missing Tree-sitter grammars from `treesit-language-source-alist'."
  (interactive)
  (dolist (grammar treesit-language-source-alist)
    (let ((lang (car grammar)))
      (unless (treesit-language-available-p lang)
        (message "Installing Tree-sitter grammar: %s" lang)
        (treesit-install-language-grammar lang)))))

(when (fboundp 'c-ts-mode)
  (add-to-list 'major-mode-remap-alist '(c-mode . c-ts-mode)))

(when (fboundp 'c++-ts-mode)
  (add-to-list 'major-mode-remap-alist '(c++-mode . c++-ts-mode)))

(when (fboundp 'python-ts-mode)
  (add-to-list 'major-mode-remap-alist '(python-mode . python-ts-mode)))

(when (fboundp 'rust-ts-mode)
  (add-to-list 'major-mode-remap-alist '(rust-mode . rust-ts-mode)))

  :mode "\\.rs\\'"
  :hook
  (rust-mode . eglot-ensure)

#+begin_src shell
emacs-power-socket() {
  printf "%s/emacs%s/power" "${TMPDIR%/}" "$(id -u)"
}

emacs-power-start() {
  local sock
  sock="$(emacs-power-socket)"

  if [ -S "$sock" ] && emacsclient -s "$sock" -e '(emacs-pid)' >/dev/null 2>&1; then
    echo "Emacs power daemon already running: $sock"
    return 0
  fi

  /Applications/Emacs.app/Contents/MacOS/Emacs \
    --init-directory "$HOME/.config/emacs-power" \
    --daemon=power \
    > /tmp/emacs-power-daemon.log 2>&1 &

  disown
}

emacs-power-gui() {
  emacsclient -s "$(emacs-power-socket)" -n -c "$@"
}

emacs-power-term() {
  emacsclient -s "$(emacs-power-socket)" -t "$@"
}

emacs-power-stop() {
  emacsclient -s "$(emacs-power-socket)" -e "(kill-emacs)"
}

emacs-power-restart() {
  emacs-power-stop 2>/dev/null || true
  sleep 1
  emacs-power-start
}

emacs-power-status() {
  pgrep -af Emacs || true
  echo "Socket: $(emacs-power-socket)"
  ls -l "$(emacs-power-socket)" 2>/dev/null || true
}

emacs-power-log() {
  tail -100 /tmp/emacs-power-daemon.log
}

emacs-power-batch-check() {
  /Applications/Emacs.app/Contents/MacOS/Emacs \
    --batch \
    --init-directory "$HOME/.config/emacs-power" \
    --eval '(message "Power config batch check OK")'
}

emacs-power-direct() {
  /Applications/Emacs.app/Contents/MacOS/Emacs \
    --init-directory "$HOME/.config/emacs-power" "$@"
}

emacs-minimal-direct() {
  /Applications/Emacs.app/Contents/MacOS/Emacs \
    --init-directory "$HOME/.config/emacs-minimal" "$@"
}
